# Spark UI — Vuetify Parity Plan

> Goal: Replicate all Vuetify 3 features in React + Tailwind + CVA.
> Strategy: Parallel subagents in worktree isolation, grouped by directory to avoid merge conflicts.

---

## How This Plan Works

Each **Phase** contains **Work Units** (WU). Each WU is designed to be executed by a single subagent in a worktree. WUs within the same phase can run **in parallel** because they touch different directories.

**Subagent workflow per WU:**
1. `isolation: "worktree"` — gets its own branch
2. Implement all items in the WU
3. Write tests + stories + a11y tests
4. Run `pnpm test --run`
5. Commit on the worktree branch
6. Return branch name → main agent merges sequentially

**After each phase:** merge all WU branches into main, resolve any conflicts, run full test suite.

## Legend

- 🆕 = New component (doesn't exist yet)
- 🔧 = Enhancement to existing component
- 🪝 = New hook
- ⚙️ = System/infrastructure feature
- `[P]` = Can be parallelized with other WUs in the same phase

## Merge Strategy

After each parallel batch:
1. Merge WU branches into main one at a time (fast-forward when possible)
2. Fix any barrel export conflicts (index.ts files)
3. Fix any a11y.test.tsx conflicts (append new tests)
4. Run `pnpm test --run` on merged result
5. Commit merge resolution
6. Proceed to next batch
