## Audit Checklist (Per Component)

Each component is reviewed against these 14 checkpoints:

| # | Check | Description |
|---|-------|-------------|
| 1 | **Semantic HTML** | Correct element types, no `<div onClick>` |
| 2 | **ARIA** | Correct roles, states, properties; live regions for dynamic content |
| 3 | **Keyboard** | Full keyboard operability; roving tabindex for composites |
| 4 | **Focus** | Visible `focus-visible:ring-2` indicator, trap in modals, restore on close |
| 5 | **Color Contrast** | 4.5:1 text, 3:1 UI; light backgrounds use dark text |
| 6 | **Reduced Motion** | All animations gated on `prefers-reduced-motion` |
| 7 | **Controlled/Uncontrolled** | Both modes work via `useControllable` where applicable |
| 8 | **SSR Safe** | `useId()` for IDs, no bare `window`/`document` access in render |
| 9 | **forwardRef + displayName** | Ref forwarding works, DevTools show name |
| 10 | **className merging** | `cn()` with consumer class last |
| 11 | **Transitions/Animations** | Smooth open/close/hover/active; enter/exit animations |
| 12 | **Performance** | No unnecessary re-renders; memo where profiled |
| 13 | **TypeScript** | Exported props types, proper generics |
| 14 | **Tests + Stories** | Unit tests, jest-axe a11y tests, Storybook stories |
