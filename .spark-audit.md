# Component Audit — CSS Variable Theming & Dark Mode

## Process (per component)

1. **Audit** — check hardcoded colors, dark mode, a11y, forwardRef, cn(), exports, contrast, stories, tests
2. **Fix** — apply changes
3. **Run tests** — verify nothing breaks
4. **Re-audit #1** — check again for anything missed
5. **Fix** (if needed)
6. **Re-audit #2** — check again
7. **Fix** (if needed)
8. **Re-audit #3** — check again
9. **Fix** (if needed)
10. **Re-audit #4** — check again
11. **Fix** (if needed)
12. **Re-audit #5** — if CLEAN → mark DONE, else fix and restart re-audit count

A component is DONE only after **5 consecutive CLEAN re-audits**.

## Checklist (each audit checks ALL of these)
- [ ] No hardcoded Tailwind colors (no `bg-red-*`, `text-slate-*`, `text-gray-*`, etc.)
- [ ] CSS variable utilities used (`bg-primary`, `text-muted-foreground`, `bg-surface`, etc.)
- [ ] Dark mode works (no `dark:` with hardcoded colors unless no CSS var exists)
- [ ] WCAG AA contrast (text/bg combos ≥ 4.5:1)
- [ ] Accessibility (ARIA, semantic HTML, focus-visible rings)
- [ ] forwardRef + displayName
- [ ] className merging via cn() with className LAST
- [ ] Proper exports (component, variants, types)
- [ ] Stories use CSS variable colors (no hardcoded `text-slate-*` in labels)
- [ ] Tests updated if class names changed
